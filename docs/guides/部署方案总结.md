## 🎯 国内访问问题

**问题**：Vercel 默认域名（`*.vercel.app`）在国内被 DNS 污染，需要挂梯子访问。

**解决方案**：使用国内可访问的托管平台。

## 📋 GitHub Pages 部署步骤

### 快速部署（推荐）

```powershell
# 1. 运行一键部署脚本
.\deploy-to-github.ps1

# 2. 在 GitHub 创建仓库
# 访问 https://github.com/new 创建新仓库

# 3. 配置远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 4. 再次运行部署脚本
.\deploy-to-github.ps1
```

### 手动部署

```bash
# 1. 创建 GitHub 仓库
# https://github.com/new

# 2. 初始化 Git
git init
git add .
git commit -m "initial commit"

# 3. 添加远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 4. 推送代码
git branch -M main
git push -u origin main

# 5. 构建项目
npm run build:github

# 6. 配置 GitHub Pages
# Settings → Pages → Source: GitHub Actions
```

## 📝 API 密钥配置建议

### 安全最佳实践

由于是纯前端应用，建议：

1. **用户自带密钥**：
   - 在设置页面让用户输入自己的 API 密钥
   - 密钥存储在浏览器本地（IndexedDB）
   - 不会上传到服务器

2. **限制密钥权限**：
   - 在 API 提供商后台设置域名白名单
   - 限制每日使用额度
   - 定期轮换密钥

3. **使用代理（可选）**：
   - Cloudflare Workers
   - Vercel Serverless Functions
   - 隐藏真实 API 密钥

## 🐛 常见问题

### 1. 页面空白或 404
**原因**：base 路径配置错误

**解决**：
```javascript
// vite.config.github.js
base: '/仓库名/', // 必须以 / 开头和结尾
```

### 2. 路由跳转后刷新 404
**原因**：GitHub Pages 不支持 History 路由

**解决**：已修改为 Hash 模式
```
https://example.com/#/calendar
```

### 3. 国内访问慢
**解决方案**：
- 使用 Gitee Pages（国内服务器）
- 绑定自定义域名 + 国内 CDN
- 使用 Cloudflare CDN 加速

### 4. API 请求失败
**原因**：CORS 跨域限制

**解决**：
- 确保 API 支持 CORS
- 配置域名白名单
- 使用代理服务器
