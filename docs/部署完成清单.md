## 📋 部署步骤

### 方式 1: 一键部署（推荐）

```powershell
# 1. 运行部署脚本
.\deploy-to-github.ps1

# 2. 在 GitHub 创建仓库
# 访问 https://github.com/new

# 3. 配置远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 4. 再次运行部署脚本
.\deploy-to-github.ps1
```

---

### 方式 2: 手动部署

```bash
# 1. 创建 GitHub 仓库
# https://github.com/new

# 2. 初始化并推送代码
git init
git add .
git commit -m "feat: 初始版本"
git remote add origin https://github.com/用户名/仓库名.git
git branch -M main
git push -u origin main

# 3. 构建项目
npm run build:github

# 4. 提交构建结果
git add .
git commit -m "build: 构建 GitHub Pages 版本"
git push origin main

# 5. 配置 GitHub Pages
# Settings → Pages → Source: GitHub Actions
```

## ⚠️ 注意事项

### 1. Base 路径配置

确保 `vite.config.js` 中的 `base` 路径与仓库名一致：

```javascript
base: '/仓库名/'
```

---

### 2. 路由使用 Hash 模式

已修改为 Hash 模式，URL 格式为：

```
https://用户名.github.io/仓库名/#/calendar
```

---

### 3. API 密钥配置

由于是纯前端应用，建议：

- ✅ 让用户在设置页面自行配置 API 密钥
- ✅ 密钥存储在浏览器本地（IndexedDB）
- ✅ 不要将密钥硬编码到代码中

---

### 4. 自定义域名（可选）

如需绑定自定义域名：

1. **DNS 配置**：
   ```
   类型：CNAME
   主机记录：www
   记录值：用户名.github.io
   ```

2. **GitHub 配置**：
   - Settings → Pages → Custom domain
   - 输入域名并保存

3. **更新 base 路径**：
   ```javascript
   base: 'https://www.yourdomain.com/'
   ```

## 🐛 常见问题

### 问题 1: 页面空白或 404

**原因**：base 路径配置错误

**解决**：
```javascript
// vite.config.js
base: '/正确的仓库名/'
```

---

### 问题 2: 刷新页面后 404

**原因**：使用了 History 路由模式

**解决**：已修改为 Hash 模式，无需额外配置

---

### 问题 3: 样式/脚本加载失败

**原因**：资源路径不正确

**解决**：
- 检查 base 路径是否正确
- 清除浏览器缓存后重试

---

### 问题 4: 国内访问慢

**解决方案**：
1. 使用 Gitee Pages（国内服务器）
2. 绑定自定义域名 + 国内 CDN
3. 使用 Cloudflare CDN 加速
